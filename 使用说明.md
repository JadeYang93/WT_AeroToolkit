# 气动组工具箱 使用说明（v0.3.11）

## 1. 简介

「气动组工具箱」是一个可扩展的工具集应用，左侧导航栏列出所有工具，右侧显示当前选中工具的操作界面。目前包含以下工具：

- **风场数据统计**：风场数据的批量统计与可视化，支持两种数据来源：
  - **秒级原始数据**：秒级时间序列文件（CSV/XLSX），自动完成风速清洗、湍流度（TI）滑动窗口计算、空气密度统计，并按日/周/月聚合出图。（从智慧风场导出时间、风速、密度秒级数据）
  - **月度湍流表**：已经按月按风速 bin 预计算好的湍流度 Excel 文件，直接出每个风速 bin 的月度时间序列曲线。（从智慧风场导出月度湍流度）

  两种数据类型**独立处理**，互不干扰——文件夹里同时存在两类文件时，按用户选择的数据类型只处理对应那一类。

- **风场对比**：把多个风场的月均风速 / 密度拉到一张图上横向对比。数据来源是各风场在「风场数据统计」模块已经生成的 `风场统计数据.xlsx`，每个风场一个子目录。聚合粒度为风场整体（跨机组简单平均）。详见 §9。

- **叶片形状输出**：三阶段风机叶片几何输出流水线（Stage1 基准翼型 → TE 修正 → Stage2 最终几何）。基于翼型库（`配置\Aerofoil_coordinate\`）与 GEO 文件（`输入数据\shape_design\*.geo`），调用 XFOIL 做尾缘厚度修正，最终输出 Focus.mac / STP 点云 / Excel 几何表等。详见 §10。

- **叶片结构套件**：FOCUS6 叶片结构分析的前后处理，覆盖 3 个子流程：
  - **TAB-1 blade_db → focus2blade**：截面属性数据库 → FOCUS6 标准 xlsx（WISDEM 插值）
  - **TAB-2 PRJ 更新**：用 focus2blade.xlsx 改写 `.prj` 项目文件的 17 个字段
  - **TAB-3 xlsx ↔ txt**：`blade_geometry.mac` ↔ `blade_data.xlsx` 双向转换

  详见 §11。

- **FOCUS6**：调用 `farob` / `frbex` / `UserLoadcaseConverter` 跑 FOCUS6 有限元分析，
  支持 7 种求解器功能 + 「一键运行」串行模式。v0.3.0 从「叶片结构套件」TAB-4 独立成新模块。
  FOCUS6 Modules 目录走「⚙ 设置」（首次使用必须先配置）。详见 §11.4。

- **载荷预估**：基准稳态/动态载荷的多项式拟合 + 新工况载荷预测。输入 `load_data.xlsx`（3 个 sheet：`baseLineSteady` / `baselineDynamic` / `newSteady`），用 6 阶多项式拟合稳态参数 → 动态载荷的映射，再代入新工况预测。输出 `result.csv` + `coefficients.csv` + 8 张 PNG。详见 §12。

- **曲线拟合**：多列离散数据 → 5 种拟合（UnivariateSpline / CubicSpline / Akima / PCHIP / 多项式）+ 任意 X 插值。源自 `F:/python/叶片前后缘光顺/pyqt_curve_fitter11.py`。自 v0.3.05 起新增「✂ 分段复用 (C2)」Tab：保留两侧原值不动，只对中段做 B 样条重设计，两个分段点处强制双端 C2 连续。自 v0.3.09 起支持**双分段点** R1/R2（如 [0, 0.5R] + [0.5R, 0.7R] 中段重设 + [0.7R, 1R] 复用）；R2=1.0 时退化为单分段点。v0.3.10 起中段控制点 X/Y 都可拖；v0.3.11 起数据输入对齐 Tab 1（载入文件 / 解析 / 清除 + 可编辑文本框，不再依赖 STAGE-1 几何表列选择）。v0.3.12 起 QTabWidget 与日志栏改为左右结构，所有解析 / 拟合 / 保存操作改写入右侧日志栏（不再弹窗）。适用于预弯 / 后掠 / 弦长 / 扭角等任何沿展向曲线的"共模设计"。详见 §13。

- **预弯设计**：用幂函数 / B 样条两种参数化方式生成叶片预弯度（prebend）分布，按用户指定的 `z_span` 数组输出。与曲线拟合共享源项目，自 v0.2.13 起剥离为独立模块。详见 §14。

- **失速评估**：由「相对厚度 → 标失速攻角」标准翼型表 + 叶片展向厚度分布，插值出每个展向位置的失速攻角，并与用户给定的实际攻角分布对比，求交点（失速起始位置）。自 v0.3.19 起支持 **VG（涡发生器）** 配置：标准翼型表加一列「VG 失速攻角」（同厚度翼型装 VG 后的失速攻角，可选），新增「VG 安装范围表」按展向子段指定哪些厚度的翼型装了 VG，对应子段内插值端点改用 VG 版攻角；VG 表空 → 退化为原 PCHIP 路径。详见 §15。

## 2. 运行环境

### 2.1 依赖

```
PyQt5            # GUI
pandas           # 数据处理
matplotlib       # 绘图
numpy            # 数值计算
openpyxl         # Excel 读写
```

安装：

```bash
pip install PyQt5 pandas matplotlib numpy openpyxl
```

### 2.2 启动

方式一（双击启动）：双击 `运行.bat`

方式二（命令行）：

```bash
python src/main.py
```

启动后显示「气动组工具箱」主界面：**左侧导航栏**列出所有工具，**右侧内容区**显示当前选中工具的面板。默认选中「风场数据统计」，右侧立即显示风场统计 UI。

风场统计面板的默认输入目录是 `项目根\输入数据\wind_farm\`、输出目录是 `项目根\输出\wind_farm\`——由全局配置中心管理（详见 §4.0 与 §8）。
**首次启动时**，若检测到旧的散落文件在 `输入数据\` 顶层、且 `config\.paths.json` 不存在，会自动把它们移入 `输入数据\wind_farm\` 子目录；以后多模块互不干扰。

## 3. 输入数据格式

把数据文件放进任一文件夹，再用本工具扫描该文件夹即可。两种数据类型可以放在同一文件夹，但**只会按当前选中的类型处理**。

### 3.1 秒级原始数据

#### 3.1.1 文件名规则（必须严格匹配）

正则：`^(.+?)_(\d+)_(.+?)_(\d{4}-\d{2}-\d{2})_(\d{4}-\d{2}-\d{2})\.(csv|xlsx)$`

模板：`风场名_机组号_机组号#_起始日期_结束日期.csv`（或 `.xlsx`）

| 段 | 含义 | 要求 |
|---|---|---|
| 风场名 | 任意文字 | 不含下划线 `_` |
| 机组号 | 整数 | 如 `010`、`21`（前导零可写可不写） |
| 机组号# | 机组号带 `#` 后缀 | 如 `010#`，仅用于展示，实际取上一段 |
| 起始日期 | 该文件代表的"当天"日期 | `YYYY-MM-DD`，**参与统计的日期以此为准** |
| 结束日期 | 次日（一般=起始+1天） | `YYYY-MM-DD`，仅用于命名规范，不参与计算 |

示例：

```
GDDLYQ_010_010#_2026-04-01_2026-04-02.csv      → 风场GDDLYQ、10号机组、当天2026-04-01
JNCYHQ_021_021#_2026-03-01_2026-03-02.xlsx     → 风场JNCYHQ、21号机组、当天2026-03-01
```

> 文件名不匹配正则的文件会被跳过，日志提示 `[警告] 跳过无法解析的文件名: xxx`。

#### 3.1.2 文件内容

**每个文件 = 一天的秒级时间序列**，约 86400 行（允许缺漏，工具按窗口有效率过滤）。至少包含**时间列**，风速列与密度列可选（顺序不固定，可只含其一、可都含）。

**列识别规则**：按列名是否**包含**关键字匹配（大小写不敏感），取**第一个匹配**的列。同一文件内某类型有多列时只认第一列。

| 列类型 | 必需 | 识别关键字（列名包含即匹配） | 数据要求 |
|---|---|---|---|
| 时间 | 必需 | `时间` / `time` / `日期` / `date` | 能被 `pd.to_datetime` 解析，如 `2026/4/1 0:00`、`2026-03-01 00:00:00` |
| 风速 | 选填 | `风速` / `wind` | 数值，单位 m/s；缺测值 ≤ -999、尖刺、负值、>50 会被清洗 |
| 密度 | 选填 | `密度` / `density` | 数值，单位 kg/m³ |

**真实示例 A**（仅含风速列，CSV，gbk 编码）：

```
time,1秒风速平均值
2026/4/1 0:00,7.13
2026/4/1 0:00,7.06
2026/4/1 0:00,7.16
...
（共 86400 行）
```
→ 识别为：时间=`time`、风速=`1秒风速平均值`、密度=无。只出风速/TI 图，不出密度图。

**真实示例 B**（仅含密度列，XLSX）：

```
time,空气密度
2026-03-01 00:00:00,1.099
2026-03-01 00:00:01,1.099
2026-03-01 00:00:02,1.099
...
（共 86259 行）
```
→ 识别为：时间=`time`、风速=无、密度=`空气密度`。只出密度图，不出风速/TI 图。

**同时含风速 + 密度**：两列都识别，三种图全出。

#### 3.1.3 CSV 编码

CSV 文件按以下顺序自动尝试解码，首个成功的生效：

```
gbk → utf-8 → utf-8-sig → gb18030
```

全部失败则跳过该文件，日志提示编码无法识别。XLSX 无编码问题。

### 3.2 月度湍流表

#### 3.2.1 文件名规则

`.xlsx` 文件，且文件名包含关键字 `月度湍流`（大小写不敏感）。

示例：

```
江华金壁_月度湍流_1781775025621.xlsx
```

#### 3.2.2 文件内容

**每行 = 一个「机组 × 月份 × 风速 bin」的湍流度记录**。一个文件可含多机组、多月份、多 bin。列名按关键字匹配，取第一个匹配项。

| 列类型 | 必需 | 识别关键字 | 数据要求 |
|---|---|---|---|
| 机组号 | 必需 | `机位` / `机组` / `涡轮` | 整数，如 `22` |
| 月份 | 必需 | `月` | 能被解析为月份，如 `2026-06-01`（datetime）或 `2026-06`（字符串）；取所属自然月 |
| 风速 bin 中心 | 必需 | `风速` / `bin` / `标签` | 整数 m/s，如 `10`（代表 9.5~10.5 m/s 段） |
| 湍流度数值 | 必需 | `数值` / `湍流` | 浮点，无量纲（TI = std/mean） |
| 样本数 | 选填 | `样本` / `数据量` / `数量` / `count` | 整数；缺失则按 1 计 |

> 多余列（如风场名、拼音码）自动忽略。无法识别齐全前四项必需列的文件会被跳过。

**真实示例**：

```
内部风场名 风场拼音码 内部机位号 月份        风速标签 样本数 湍流
江华金壁   JHJBFC    22        2026-06-01  10      7      0.173
江华金壁   JHJBFC    23        2026-06-01  10      6      0.192
江华金壁   JHJBFC    24        2026-06-01  10      27     0.188
...
（共 207 行，覆盖机组 22~28、月份 2024-01~2026-06、bin 10）
```
→ 识别为：机组号=`内部机位号`、月份=`月份`、bin=`风速标签`、TI=`湍流`、样本=`样本数`。`内部风场名` / `风场拼音码` 忽略。

## 4. 界面操作 — 风场数据统计面板

主界面分左右两区：**左侧导航栏**（窄列，列出工具集所有工具）+ **右侧内容区**（当前选中工具的完整操作 UI）。点左侧条目切换右侧内容。

下面的小节描述**「风场数据统计」面板**内部的控件，从上到下依次为。「风场对比」面板的结构与之类似（banner + 左右双列 + 底部执行栏），具体差异见 §9。

### 4.0 左侧导航栏

- 控件：`QListWidget`，固定宽 172px，纯文字单列
- 当前条目：「🌬 风场数据统计」「⚖ 风场对比」「✈ 叶片形状输出」「🔧 叶片结构套件」「📊 载荷预估」「📈 曲线拟合」「📐 预弯设计」共 7 项
- 启动后默认选中第 1 项，右侧立即显示对应面板
- 未来加新工具时，此栏会自动多出条目（详见 §8 目录结构与「加新工具的步骤」）
- **底部「⚙ 设置」按钮**：弹出路径设置对话框（§4.7），是**唯一**能持久化修改输入/输出目录的入口

### 4.1 数据文件夹

- **数据类型**：下拉选择 `秒级原始数据` 或 `月度湍流表`。切换后会自动重新扫描。
- **路径**：当前数据文件夹绝对路径（来自全局配置中心，初始为 `输入数据\wind_farm\`）。
- **扫描**：手动扫描当前文件夹下选中类型的文件，刷新机组列表与日期范围。**启动时不自动扫描**，必须先点此按钮或切换数据类型才能加载机组列表。
- **浏览...**：弹出对话框选择其他文件夹，选完自动扫描。**仅本次会话生效，重启后恢复**；要永久修改路径请用「⚙ 设置」（§4.7）。

扫描结果会显示在路径下方一行，包括：找到的文件数、机组数及机组号列表、日期范围（同时设置日期选择控件的可选上下限）。

### 4.2 统计选项（仅"秒级原始数据"模式有效）

- **指标**：风速 / 湍流度 / 密度，多选，至少勾 1 个。
- **粒度**：日 / 周 / 月，多选，至少勾 1 个。
- **机组表**：四列依次为 **机组号 / 包含 / 高亮 / 叶型**。
  - **包含**：默认全部勾选；可手动取消某些机组跳过统计。提供 **全选 / 全不选** 快捷按钮。至少保留 1 个。
  - **高亮**：默认全不勾；勾选后该机组在曲线图中线宽加粗、不透明度提高，其他机组变淡以突出对比。提供 **全选 / 全不选** 快捷按钮。
  - **叶型**：手动输入文本，可留空。用于按叶型在图中区分机组线型与图例。详见 4.3。
  - 机组号列与各复选框不可编辑；仅叶型列可双击或按 F2 编辑。

### 4.3 叶型-线型映射

机组表下方为 **叶型-线型映射** 表（标题旁带「刷新」按钮），用于为每种叶型指定图中曲线的线型。

- **触发刷新**：点 **刷新** 按钮或点 **开 始 统 计** 时自动刷新一次。
- **表行来源**：自动收集机组表叶型列中出现过的所有名称（去重）；只要存在任意一台机组叶型留空，就会多出一行 `(未指定)`。
- **线型选项**：下拉框可选 `实线 / 虚线 / 点划线 / 点线`。默认按行序自动分配（第 1 行实线，第 2 行虚线，依此类推，超出选项数后取最后一项）。
- **图例与线型规则**：
  - 已填叶型的机组：曲线使用该叶型对应线型；图例为 `#N号机组 (叶型名)`（N 为机组号，如 `#22号机组 (A)`）。
  - 未填叶型的机组：曲线使用 `(未指定)` 行对应的线型；图例为 `#N号机组`（无括号后缀）。
  - 线型只与叶型绑定，颜色仍按机组号自动分配，两者互不影响。
- **单机组图标题**：每个机组单独成图的输出（如 `机组{t}号...png`），标题后会追加 `[叶型: X]`（未填则不追加）。
- **跨扫描保留**：重新扫描机组列表时，已填写的叶型文本与已选线型会保留；删除的机组不再参与。

### 4.4 湍流度计算参数（仅"秒级原始数据"模式有效）

| 参数 | 含义 | 默认 | 范围 |
|---|---|---|---|
| 滑动窗口（分钟） | TI 计算窗口长度 | 10 | 1 ~ 60 |
| 滑动步长（分钟） | 窗口移动步长 | 1 | 1 ~ 30 |
| 有效率（%） | 窗口内有效数据占比阈值，低于则该窗口不计 TI | 90 | 0 ~ 100 |

### 4.5 日期范围

- 勾选 **全部日期**：使用扫描到的全部日期（默认）。
- 取消勾选：手动选择起止日期（受扫描结果约束的可选范围）。

### 4.6 开始统计

点 **开 始 统 计** 按钮启动后台线程处理。按钮在处理期间变灰，下方进度条与日志区实时刷新。处理完成后日志区显示生成图数与 Excel 路径。

### 4.7 路径设置（导航栏底部「⚙ 设置」）

打开后是一个**表格**对话框，每行一个模块，列出其**输入目录**与**输出目录**：

- 修改后点 **确定**：执行三层校验（输入必须存在且可读、输出父目录必须可写）→ 通过则持久化到 `config\.paths.json`，并广播给所有面板立即刷新显示 + 自动重新扫描；失败则列出错误、**对话框保持打开**让你逐条修。
- 点 **取消**：丢弃所有改动。
- 面板内的「浏览...」按钮改路径只是**临时**会话切换，重启后恢复到这里的持久化值。

> 该对话框是修改路径的**唯一持久化入口**。直接编辑 `.paths.json` 也可以，但下次启动如果 JSON 损坏，会被自动备份为 `.paths.json.corrupt-{时间戳}` 并重建空配置。

**模块额外路径（extras）**：表格下方按模块分组显示各模块的额外配置：
- **FOCUS6**：「FOCUS6 Modules 目录」（内含 `farob/` / `frbex/` / `utils/` 等子目录）

> 注：「叶片形状输出」的翼型库目录自 v0.3.03 起固定为工具箱根目录下的 `配置\`，不再作为 extras 显示（跨电脑迁移时自动跟随工具箱根目录）。

校验规则：必须是已存在的目录。留空表示未配置（FOCUS6 面板顶部会显示「(未配置)」并拒绝运行）。详见 §11.5。

> v0.3.0 起，`modules_path` 已从「叶片结构套件」迁到「FOCUS6」；首次启动 v0.3.0
> 时，旧配置会自动复制到新模块（一次性，不删源）。

## 5. 输出结果

所有输出文件写入当前模块的输出目录（默认 `输出\wind_farm\`，可在「⚙ 设置」中改），运行时自动创建。同目录下的 `.cache.pkl` 为读取缓存（按文件 mtime/size + TI 参数 + 缓存版本号作键），用于加速重复扫描，可安全删除。

## 6. 处理逻辑说明

### 6.1 风速清洗（秒级原始数据模式，4 步）

1. 缺测值（≤ -999）置 NaN
2. 前后秒差 > 10 m/s 视为尖刺，置 NaN
3. 负值置 NaN
4. 超量程（> 50 m/s）置 NaN

### 6.2 TI 计算（秒级原始数据模式）

- 按设定的窗口/步长在清洗后的风速序列上滑动
- 窗口有效率达标 → 计算该窗口的 TI = std / mean（要求平均风速 > 0.5 m/s）
- 日均 TI = 当天所有有效窗口 TI 的均值

### 6.3 周/月聚合阈值

- 某机组某周数据天数 < 5 → 不出该周曲线
- 某机组某月数据天数 < 5 → 不出该月曲线

## 7. 常见问题

| 现象 | 可能原因与处理 |
|---|---|
| 启动后机组列表为空 | 默认未自动扫描，请先点「扫描」或切换数据类型 |
| 提示"未找到符合命名规则的数据文件" | 检查文件名是否严格匹配正则 `^(.+?)_(\d+)_(.+?)_(\d{4}-\d{2}-\d{2})_(\d{4}-\d{2}-\d{2})\.(csv\|xlsx)$` |
| 提示"未找到月度湍流文件" | 文件名需含 `月度湍流` 且为 `.xlsx` 后缀 |
| 某些文件被跳过 | 日志会标注原因：编码无法识别 / 无时间列 / 无风速列等 |
| 月/周曲线缺失某些机组 | 该机组该月/周有效天数 < 5 天，自动剔除 |
| 切换数据类型后日期范围不更新 | 数据类型切换会自动触发重扫；若仍未更新，手动点「扫描」 |
| 月度湍流模式勾选了风速/密度却没出图 | 此模式只输出月度 TI 时间序列，指标勾选在此模式下被忽略 |
| 缓存导致结果异常 | 删除 `输出\wind_farm\.cache.pkl` 后重新运行 |
| **风场对比** 扫描显示「缺 Excel: 风场X」 | 该子目录缺 `风场统计数据.xlsx`；从「风场数据统计」模块的输出目录拷过来 |
| **风场对比** 日志提示「缺 月均风速 sheet」 | 该 xlsx 是在「风场数据统计」未勾选「月」粒度时生成的，需重跑并勾选「月」 |
| **风场对比** 图里某风场只有部分月份 | 该风场数据本身覆盖范围短（如只有 5-12 月），符合预期，不补 0 |

## 9. 风场对比模块（wind_farm_compare）

把多个风场的月均风速 / 密度拉到一张图上横向对比，看哪个风场风况更好、密度差异等。

### 9.1 输入约定

输入目录（默认 `输入数据\wind_farm_compare\`）下**每个子目录 = 一个风场**：

```
输入数据\wind_farm_compare\
├── 风场A\
│   └── 风场统计数据.xlsx      ← 从「风场数据统计」模块的输出目录拷过来
├── 风场B\
│   └── 风场统计数据.xlsx
└── 风场C\
    └── 风场统计数据.xlsx
```

- **子目录名 = 风场名**：作为图例和 Excel 列名直接显示
- **必须包含 `风场统计数据.xlsx`**：由「风场数据统计」模块生成
- **Sheet 名要求**：`月均风速` / `月均密度`（也兼容 `月平均风速` / `月平均密度`）
- 隐藏目录（`.` 开头）会被忽略

> ⚠️ **重要**：`风场统计数据.xlsx` 必须在「风场数据统计」模块勾选**「月」粒度**的情况下生成，否则会缺月均 sheet。

### 9.2 聚合方式

**风场整体对比**（不是按机组）：

1. 取该风场 `月均风速` sheet 中所有机组的月均
2. 按月份 `groupby` 后简单平均，得到「该风场该月的全风场均值」
3. 每个风场一条线

## 12. 载荷预估模块（load_estimation）

### 12.1 功能

基准稳态/动态载荷的多项式拟合 + 新工况载荷预测。把一组基准工况（稳态参数 + 动态载荷）
和一组新工况（只有稳态参数）合并分析，用 6 阶多项式拟合「稳态参数 → 动态载荷」映射，
再把新工况稳态参数代入预测。

源项目 `F:/python/载荷信息读取/load_estimation.py`（Tkinter）已重写为 PyQt5。

### 12.2 输入数据

放一份 `load_data.xlsx` 到 `输入数据\load_estimation\`，必须包含 3 个 sheet：

| Sheet 名 | 列含义 |
|---|---|
| `baseLineSteady` | `t, paramX, paramY, ...`（基准稳态：t 是展向位置） |
| `baselineDynamic` | `t, maxMx, minMx, maxMy, minMy`（基准动态：4 个载荷分量） |
| `newSteady` | `t, paramX, paramY, ...`（新工况稳态） |

第 1 行为表头（自动跳过）。`baseLineSteady` 和 `baselineDynamic` 行数应相等。

### 12.3 拟合关系

- `maxMx / minMx ← polyfit(baseLineSteady.paramX, baselineDynamic.maxMx / minMx, 6)`
- `maxMy / minMy ← polyfit(baseLineSteady.paramY, baselineDynamic.maxMy / minMy, 6)`
- 新工况预测：把 `newSteady.paramX / paramY` 代入上面的 6 阶多项式

### 12.4 输出

全部写入 `输出\load_estimation\`：

| 文件 | 内容 |
|---|---|
| `result.csv` | 新工况预测结果（zspan, maxMx, minMx, maxMy, minMy） |
| `coefficients.csv` | 4 组多项式系数（高次 → 低次） |
| `figure_baseline_*.png` × 4 | 基准动态：原始 vs 拟合曲线 |
| `figure_new_*.png` × 4 | 新工况预测曲线 |

### 12.5 UI 操作

1. 「数据文件」浏览选择 `load_data.xlsx`
2. 点「运行拟合」→ 后台线程执行读取 + 拟合 + 写文件
3. 完成后，「拟合曲线」区下拉框激活，可选 8 个视图（baseline × 4 + new × 4）
4. 用「上一张 / 下一张」顺序翻看
5. 「📂 打开输出目录」直接打开 `输出\load_estimation\`

## 14. 预弯设计模块（prebend_design）

### 14.1 功能

用两种参数化方式生成叶片预弯度（prebend）沿展向的分布，按用户指定的 `z_span` 输出对应值。
自 v0.2.13 从原 curve_fitter 模块剥离为独立模块，业务逻辑位于 `src/prebend_design/`。

> v0.3.06 起：「约束 B 样条」模式移除（与 B 样条重叠且使用率低）；B 样条模式的第二个控制点（idx 1）从 `y=0` 锁定改为完全自由拖动。
>
> v0.3.07 起：拖动交互改进——悬停手形光标（提示可拖）/ 按下握拳 / 黄色高亮圆（显示正在拖哪个 idx）/ 命中半径从 12 提升到 20 像素，中间控制点更容易点中。

### 14.2 两种模式

| 模式 | 关键参数 | 适用场景 |
|---|---|---|
| 幂函数 (Power) | 桨尖预弯 `tip_pb`、起始位置比 `z/R`、指数 `γ` | 经验公式快速生成 |
| B 样条 (Bspline) | 连续阶（C0/C1/C2）、控制点表 | 已知控制点，生成光滑曲线 |

B 样条模式控制点：绿色方块（首点，y=0 锁定）/ 红色圆点（中间，自由拖动）/ 蓝色三角（末点，x=1.0 锁定）。

公共参数：`z_span`（展向坐标数组，默认 `np.linspace(0, 80, 54)`）可编辑。

### 14.3 操作

1. 切模式 → 填参数（B 样条模式还要填控制点表，可用「+ 添加」/「− 删除」或双击画布新增）

## 15. 失速评估模块（stall_assessment）

### 15.1 功能

由「相对厚度 → 标失速攻角」的标准翼型表 + 叶片展向厚度分布，用 PCHIP 保形插值
求出叶片沿展向各站的失速攻角分布，并与用户给定的最大攻角分布对比，求交点
（标出失速起始位置）。

自 v0.3.19 起支持 **VG（涡发生器）** 配置：

- 标准翼型表加一列「VG 失速攻角」——同一种厚度的翼型，装 VG 后失速攻角会升高
  （如 30% 厚度无 VG 是 13°，有 VG 是 15°）。VG 列空 = 该厚度无 VG 变体。
- 新增「VG 安装范围表」——每行描述「哪个标准厚度的翼型在哪个 z 子段内装了 VG」
  （标准厚度下拉 + VG 起/止 z）。
- 装了 VG 的子段，插值时该厚度的失速攻角端点改用 VG 版；VG 表空 → 退化为
  原 PCHIP 路径（结果与无 VG 版完全一致）。

### 15.2 失速评估 SOP（标准操作流程）

完成一次失速评估的整体流程：前 5 步在 Bladed 中准备「最大攻角沿展向分布」，
第 6 步回到本模块完成插值与对比。点面板上的「📖 失速评估 SOP」可随时查看。

| # | 操作 | 说明 |
|---|---|---|
| 1 | pcoeffs 计算 | 在 Bladed 中运行 pcoeffs 计算，得到各风速下的功率系数。 |
| 2 | control k_opt 更新 | 更新最优控制增益 k_opt（变量桨距增益），保证后续稳态计算的桨距角跟踪最优。 |
| 3 | steadyop → Calculation Output → Aerodynamic Information → First Blade | 在 steadyop（稳态计算）中，将 Calculation Output 的 Aerodynamic Information 输出项设为 First Blade。 |
| 4 | data view → steadyop → Summary Info → Electrical Power → 找额定功率对应风速 − 0.1 | 在 Data View 中查看 steadyop 的 Summary Info，定位 Electrical Power 等于额定功率对应的风速；该风速减 0.1 m/s 通常就是攻角最大的风速（可在 ±0.1° 桨距角下对比攻角以验证）。 |
| 5 | steadyop → Blade 1 Information → Angle of Attack → 对应风速下的攻角分布 | 在 steadyop 的 Blade 1 Information 中取 Angle of Attack，导出"上一步定位的风速"对应的沿展向攻角分布。 |
| 6 | 标准翼型失速攻角 + 相对厚度插值 + 与最大攻角对比（本模块） | 将"标准翼型表"填入左上表，配入展向厚度分布与第 5 步得到的攻角分布，点击「计算失速攻角」——本模块自动按相对厚度插值求出各站失速攻角，并与最大攻角对比、标注失速起始位置。 |

### 15.3 输入数据

四组数据，都在面板里直接给出：

1. **标准翼型表**（左上表格）：三列「相对厚度 / 标失速攻角 / VG 失速攻角（可选）」，
   可增删行。
2. **展向分布**：每行「展向位置 r/R, 相对厚度」，可粘贴或载入 CSV/xlsx。
3. **攻角分布**：每行「展向位置 r/R, 最大攻角°」，用于与失速攻角对比求交点。
4. **VG 安装范围表**（可选）：每行「标准厚度（下拉，来自 VG 列非空的项）+
   VG 起/止 z」，可增删行。空表 = 不使用 VG。

> 相对厚度单位需一致（同用 0.18 或 18，模块不换算）。

### 15.4 插值方法

**默认路径（无 VG）**：PCHIP 保形插值，单调、不过冲，适合失速攻角这类物理量。

**VG 路径**：对每个展向位置 z：

1. 找当前相对厚度 t(z) 落在哪两个相邻标准厚度之间（区间 [t_a, t_b]）。
2. 分别决定 t_a、t_b 在 z 处用标攻角还是 VG 攻角：z 在该厚度的 VG 安装段内 → α_VG，
   否则 → α_标。
3. 在 [t_a, t_b] 内对当前 t(z) 做两点线性插值。

**为什么 VG 段用两点线性而不是 PCHIP？** PCHIP 是全局方法，VG 只改一个端点会"涟漪"
扩散到全曲线；两点线性保证 VG 影响严格限制在涉及该厚度端点的子段内，符合
"VG 只影响局部"的物理直觉。

**连续性**：段内必然连续；段边界自动连续；VG 装卸边界处攻角会**跳变**——这是
VG 终止的物理表现，符合预期（非 bug）。

**超范围**：相对厚度超出标准表范围（如叶根 1.0 但标准表只到 0.6）→ 报错提示
补全标准表，不做外推。

### 15.5 输出

`输出\stall_assessment\stall_alpha_span.csv`：

- 无 VG：三列 `span_position, relative_thickness, stall_alpha_deg`
- 有 VG：追加 `vg_active` 列（1.0/0.0，该 z 是否落在任何 VG 段内）

面板右侧同时显示失速攻角展向分布图（含 VG 段橙色阴影 + 边界灰色竖虚线 + 失速
攻角与最大攻角对比 + 交点标注）+ 结果表（可复制）。
