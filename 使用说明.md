# 气动组工具箱 使用说明（v0.1）

## 1. 简介

「气动组工具箱」是一个可扩展的工具集应用，左侧导航栏列出所有工具，右侧显示当前选中工具的操作界面。目前包含以下工具：

- **风场数据统计**：风场数据的批量统计与可视化，支持两种数据来源：
  - **秒级原始数据**：秒级时间序列文件（CSV/XLSX），自动完成风速清洗、湍流度（TI）滑动窗口计算、空气密度统计，并按日/周/月聚合出图。（从智慧风场导出时间、风速、密度秒级数据）
  - **月度湍流表**：已经按月按风速 bin 预计算好的湍流度 Excel 文件，直接出每个风速 bin 的月度时间序列曲线。（从智慧风场导出月度湍流度）

  两种数据类型**独立处理**，互不干扰——文件夹里同时存在两类文件时，按用户选择的数据类型只处理对应那一类。

- **风场对比**：把多个风场的月均风速 / 密度拉到一张图上横向对比。数据来源是各风场在「风场数据统计」模块已经生成的 `风场统计数据.xlsx`，每个风场一个子目录。聚合粒度为风场整体（跨机组简单平均）。详见 §9。

- **叶片形状输出**：三阶段风机叶片几何输出流水线（Stage1 基准翼型 → TE 修正 → Stage2 最终几何）。基于 AppData 描述文件与 GEO 文件，调用 XFOIL 做尾缘厚度修正，最终输出 Focus.mac / STP 点云 / Excel 几何表等。详见 §10。

- **叶片结构套件**：FOCUS6 叶片结构分析的一站式前后处理，覆盖 4 个子流程：
  - **TAB-1 blade_db → focus2blade**：截面属性数据库 → FOCUS6 标准 xlsx（WISDEM 插值）
  - **TAB-2 PRJ 更新**：用 focus2blade.xlsx 改写 `.prj` 项目文件的 17 个字段
  - **TAB-3 Excel ↔ txt**：`blade_geometry.mac` ↔ `blade_data.xlsx` 双向转换
  - **TAB-4 FOCUS6 求解器**：调用 `farob` / `frbex` / `UserLoadcaseConverter`，支持「一键运行」

  3 个求解器 `.exe` 路径走「⚙ 设置」（首次使用必须先配置）。详见 §11。

## 2. 运行环境

### 2.1 依赖

```
PyQt5            # GUI
pandas           # 数据处理
matplotlib       # 绘图
numpy            # 数值计算
openpyxl         # Excel 读写
```

安装：

```bash
pip install PyQt5 pandas matplotlib numpy openpyxl
```

### 2.2 启动

方式一（双击启动）：双击 `运行.bat`

方式二（命令行）：

```bash
python src/main.py
```

启动后显示「气动组工具箱」主界面：**左侧导航栏**列出所有工具，**右侧内容区**显示当前选中工具的面板。默认选中「风场数据统计」，右侧立即显示风场统计 UI。

风场统计面板的默认输入目录是 `项目根\输入数据\wind_farm\`、输出目录是 `项目根\输出\wind_farm\`——由全局配置中心管理（详见 §4.0 与 §8）。
**首次启动时**，若检测到旧的散落文件在 `输入数据\` 顶层、且 `config\.paths.json` 不存在，会自动把它们移入 `输入数据\wind_farm\` 子目录；以后多模块互不干扰。

## 3. 输入数据格式

把数据文件放进任一文件夹，再用本工具扫描该文件夹即可。两种数据类型可以放在同一文件夹，但**只会按当前选中的类型处理**。

### 3.1 秒级原始数据

#### 3.1.1 文件名规则（必须严格匹配）

正则：`^(.+?)_(\d+)_(.+?)_(\d{4}-\d{2}-\d{2})_(\d{4}-\d{2}-\d{2})\.(csv|xlsx)$`

模板：`风场名_机组号_机组号#_起始日期_结束日期.csv`（或 `.xlsx`）

| 段 | 含义 | 要求 |
|---|---|---|
| 风场名 | 任意文字 | 不含下划线 `_` |
| 机组号 | 整数 | 如 `010`、`21`（前导零可写可不写） |
| 机组号# | 机组号带 `#` 后缀 | 如 `010#`，仅用于展示，实际取上一段 |
| 起始日期 | 该文件代表的"当天"日期 | `YYYY-MM-DD`，**参与统计的日期以此为准** |
| 结束日期 | 次日（一般=起始+1天） | `YYYY-MM-DD`，仅用于命名规范，不参与计算 |

示例：

```
GDDLYQ_010_010#_2026-04-01_2026-04-02.csv      → 风场GDDLYQ、10号机组、当天2026-04-01
JNCYHQ_021_021#_2026-03-01_2026-03-02.xlsx     → 风场JNCYHQ、21号机组、当天2026-03-01
```

> 文件名不匹配正则的文件会被跳过，日志提示 `[警告] 跳过无法解析的文件名: xxx`。

#### 3.1.2 文件内容

**每个文件 = 一天的秒级时间序列**，约 86400 行（允许缺漏，工具按窗口有效率过滤）。至少包含**时间列**，风速列与密度列可选（顺序不固定，可只含其一、可都含）。

**列识别规则**：按列名是否**包含**关键字匹配（大小写不敏感），取**第一个匹配**的列。同一文件内某类型有多列时只认第一列。

| 列类型 | 必需 | 识别关键字（列名包含即匹配） | 数据要求 |
|---|---|---|---|
| 时间 | 必需 | `时间` / `time` / `日期` / `date` | 能被 `pd.to_datetime` 解析，如 `2026/4/1 0:00`、`2026-03-01 00:00:00` |
| 风速 | 选填 | `风速` / `wind` | 数值，单位 m/s；缺测值 ≤ -999、尖刺、负值、>50 会被清洗 |
| 密度 | 选填 | `密度` / `density` | 数值，单位 kg/m³ |

**真实示例 A**（仅含风速列，CSV，gbk 编码）：

```
time,1秒风速平均值
2026/4/1 0:00,7.13
2026/4/1 0:00,7.06
2026/4/1 0:00,7.16
...
（共 86400 行）
```
→ 识别为：时间=`time`、风速=`1秒风速平均值`、密度=无。只出风速/TI 图，不出密度图。

**真实示例 B**（仅含密度列，XLSX）：

```
time,空气密度
2026-03-01 00:00:00,1.099
2026-03-01 00:00:01,1.099
2026-03-01 00:00:02,1.099
...
（共 86259 行）
```
→ 识别为：时间=`time`、风速=无、密度=`空气密度`。只出密度图，不出风速/TI 图。

**同时含风速 + 密度**：两列都识别，三种图全出。

#### 3.1.3 CSV 编码

CSV 文件按以下顺序自动尝试解码，首个成功的生效：

```
gbk → utf-8 → utf-8-sig → gb18030
```

全部失败则跳过该文件，日志提示编码无法识别。XLSX 无编码问题。

### 3.2 月度湍流表

#### 3.2.1 文件名规则

`.xlsx` 文件，且文件名包含关键字 `月度湍流`（大小写不敏感）。

示例：

```
江华金壁_月度湍流_1781775025621.xlsx
```

#### 3.2.2 文件内容

**每行 = 一个「机组 × 月份 × 风速 bin」的湍流度记录**。一个文件可含多机组、多月份、多 bin。列名按关键字匹配，取第一个匹配项。

| 列类型 | 必需 | 识别关键字 | 数据要求 |
|---|---|---|---|
| 机组号 | 必需 | `机位` / `机组` / `涡轮` | 整数，如 `22` |
| 月份 | 必需 | `月` | 能被解析为月份，如 `2026-06-01`（datetime）或 `2026-06`（字符串）；取所属自然月 |
| 风速 bin 中心 | 必需 | `风速` / `bin` / `标签` | 整数 m/s，如 `10`（代表 9.5~10.5 m/s 段） |
| 湍流度数值 | 必需 | `数值` / `湍流` | 浮点，无量纲（TI = std/mean） |
| 样本数 | 选填 | `样本` / `数据量` / `数量` / `count` | 整数；缺失则按 1 计 |

> 多余列（如风场名、拼音码）自动忽略。无法识别齐全前四项必需列的文件会被跳过。

**真实示例**：

```
内部风场名 风场拼音码 内部机位号 月份        风速标签 样本数 湍流
江华金壁   JHJBFC    22        2026-06-01  10      7      0.173
江华金壁   JHJBFC    23        2026-06-01  10      6      0.192
江华金壁   JHJBFC    24        2026-06-01  10      27     0.188
...
（共 207 行，覆盖机组 22~28、月份 2024-01~2026-06、bin 10）
```
→ 识别为：机组号=`内部机位号`、月份=`月份`、bin=`风速标签`、TI=`湍流`、样本=`样本数`。`内部风场名` / `风场拼音码` 忽略。

## 4. 界面操作 — 风场数据统计面板

主界面分左右两区：**左侧导航栏**（窄列，列出工具集所有工具）+ **右侧内容区**（当前选中工具的完整操作 UI）。点左侧条目切换右侧内容。

下面的小节描述**「风场数据统计」面板**内部的控件，从上到下依次为。「风场对比」面板的结构与之类似（banner + 左右双列 + 底部执行栏），具体差异见 §9。

### 4.0 左侧导航栏

- 控件：`QListWidget`，固定宽 172px，纯文字单列
- 当前条目：「🌬 风场数据统计」「⚖ 风场对比」「✈ 叶片形状输出」「🔧 叶片结构套件」共 4 项
- 启动后默认选中第 1 项，右侧立即显示对应面板
- 未来加新工具时，此栏会自动多出条目（详见 §8 目录结构与「加新工具的步骤」）
- **底部「⚙ 设置」按钮**：弹出路径设置对话框（§4.7），是**唯一**能持久化修改输入/输出目录的入口

### 4.1 数据文件夹

- **数据类型**：下拉选择 `秒级原始数据` 或 `月度湍流表`。切换后会自动重新扫描。
- **路径**：当前数据文件夹绝对路径（来自全局配置中心，初始为 `输入数据\wind_farm\`）。
- **扫描**：手动扫描当前文件夹下选中类型的文件，刷新机组列表与日期范围。**启动时不自动扫描**，必须先点此按钮或切换数据类型才能加载机组列表。
- **浏览...**：弹出对话框选择其他文件夹，选完自动扫描。**仅本次会话生效，重启后恢复**；要永久修改路径请用「⚙ 设置」（§4.7）。

扫描结果会显示在路径下方一行，包括：找到的文件数、机组数及机组号列表、日期范围（同时设置日期选择控件的可选上下限）。

### 4.2 统计选项（仅"秒级原始数据"模式有效）

- **指标**：风速 / 湍流度 / 密度，多选，至少勾 1 个。
- **粒度**：日 / 周 / 月，多选，至少勾 1 个。
- **机组表**：四列依次为 **机组号 / 包含 / 高亮 / 叶型**。
  - **包含**：默认全部勾选；可手动取消某些机组跳过统计。提供 **全选 / 全不选** 快捷按钮。至少保留 1 个。
  - **高亮**：默认全不勾；勾选后该机组在曲线图中线宽加粗、不透明度提高，其他机组变淡以突出对比。提供 **全选 / 全不选** 快捷按钮。
  - **叶型**：手动输入文本，可留空。用于按叶型在图中区分机组线型与图例。详见 4.3。
  - 机组号列与各复选框不可编辑；仅叶型列可双击或按 F2 编辑。

### 4.3 叶型-线型映射

机组表下方为 **叶型-线型映射** 表（标题旁带「刷新」按钮），用于为每种叶型指定图中曲线的线型。

- **触发刷新**：点 **刷新** 按钮或点 **开 始 统 计** 时自动刷新一次。
- **表行来源**：自动收集机组表叶型列中出现过的所有名称（去重）；只要存在任意一台机组叶型留空，就会多出一行 `(未指定)`。
- **线型选项**：下拉框可选 `实线 / 虚线 / 点划线 / 点线`。默认按行序自动分配（第 1 行实线，第 2 行虚线，依此类推，超出选项数后取最后一项）。
- **图例与线型规则**：
  - 已填叶型的机组：曲线使用该叶型对应线型；图例为 `#N号机组 (叶型名)`（N 为机组号，如 `#22号机组 (A)`）。
  - 未填叶型的机组：曲线使用 `(未指定)` 行对应的线型；图例为 `#N号机组`（无括号后缀）。
  - 线型只与叶型绑定，颜色仍按机组号自动分配，两者互不影响。
- **单机组图标题**：每个机组单独成图的输出（如 `机组{t}号...png`），标题后会追加 `[叶型: X]`（未填则不追加）。
- **跨扫描保留**：重新扫描机组列表时，已填写的叶型文本与已选线型会保留；删除的机组不再参与。

### 4.4 湍流度计算参数（仅"秒级原始数据"模式有效）

| 参数 | 含义 | 默认 | 范围 |
|---|---|---|---|
| 滑动窗口（分钟） | TI 计算窗口长度 | 10 | 1 ~ 60 |
| 滑动步长（分钟） | 窗口移动步长 | 1 | 1 ~ 30 |
| 有效率（%） | 窗口内有效数据占比阈值，低于则该窗口不计 TI | 90 | 0 ~ 100 |

### 4.5 日期范围

- 勾选 **全部日期**：使用扫描到的全部日期（默认）。
- 取消勾选：手动选择起止日期（受扫描结果约束的可选范围）。

### 4.6 开始统计

点 **开 始 统 计** 按钮启动后台线程处理。按钮在处理期间变灰，下方进度条与日志区实时刷新。处理完成后日志区显示生成图数与 Excel 路径。

### 4.7 路径设置（导航栏底部「⚙ 设置」）

打开后是一个**表格**对话框，每行一个模块，列出其**输入目录**与**输出目录**：

- 修改后点 **确定**：执行三层校验（输入必须存在且可读、输出父目录必须可写）→ 通过则持久化到 `config\.paths.json`，并广播给所有面板立即刷新显示 + 自动重新扫描；失败则列出错误、**对话框保持打开**让你逐条修。
- 点 **取消**：丢弃所有改动。
- 面板内的「浏览...」按钮改路径只是**临时**会话切换，重启后恢复到这里的持久化值。

> 该对话框是修改路径的**唯一持久化入口**。直接编辑 `.paths.json` 也可以，但下次启动如果 JSON 损坏，会被自动备份为 `.paths.json.corrupt-{时间戳}` 并重建空配置。

**叶片结构套件模块的求解器路径**：表格下方额外有一个 **「叶片结构套件 — 求解器路径」** 分组，配置 3 个 FOCUS6 `.exe`（`farob` / `frbex` / `UserLoadcaseConverter`）。校验规则：必须是 `.exe` 后缀 + 文件本身存在。留空表示未配置（TAB-4 顶部会显示「(未配置)」并拒绝运行）。详见 §11.2。

## 5. 输出结果

所有输出文件写入当前模块的输出目录（默认 `输出\wind_farm\`，可在「⚙ 设置」中改），运行时自动创建。同目录下的 `.cache.pkl` 为读取缓存（按文件 mtime/size + TI 参数 + 缓存版本号作键），用于加速重复扫描，可安全删除。

> **叶型对图例的影响（两种模式通用）**：若在机组表填写了叶型并配置了线型映射，所有多机组对比图的曲线线型与图例文字会按 4.3 的规则改变；每个机组单独成图的标题会追加 `[叶型: X]`。未填叶型时不影响任何输出。

### 5.1 秒级原始数据模式输出

**曲线图**（PNG，按勾选的指标 × 粒度生成，最多 9 张）：

| 指标 | 日 | 周 | 月 |
|---|---|---|---|
| 风速 | 日平均风速曲线.png | 周平均风速曲线.png | 月平均风速曲线.png |
| 湍流度 | 日平均湍流度曲线.png | 周平均湍流度曲线.png | 月平均湍流度曲线.png |
| 密度 | 日平均空气密度曲线.png | 周平均空气密度曲线.png | 月平均空气密度曲线.png |

**湍流度-风速 bin P90 分布曲线**（仅勾选湍流度时生成）：

- `湍流度-风速分布P90曲线.png`（全期）
- `湍流度-风速分布P90曲线_YYYY-MM.png`（按月，需勾选月粒度）
- `湍流度-风速分布P90曲线_周YYYY-MM-DD.png`（按周，需勾选周粒度）

bin 划分规则：1 m/s 宽度、中心取整数（9.5~10.5 m/s → bin 10）；每个 bin 至少 10 个有效窗口才计算 P90。

这三张图均为**上下双轴**布局：上方是 P90 TI-风速 bin 曲线（每机组一条线），下方是**有效窗口数分组柱状图**（每个 bin 内每机组一根柱，颜色与上方曲线一一对应），便于判断各 bin 的样本可靠度。

**每机组 P90 TI-风速bin 曲线（按月对比）**（仅勾选「湍流度」+「月」粒度时生成）：

- `机组{t}号湍流度-风速分布P90曲线.png`（如 `机组10号湍流度-风速分布P90曲线.png`）

  **每台机组一张图**，横轴为风速 bin，纵轴为 P90 湍流度，**每条线 = 一个月**。与按月份分图（`..._2026-04.png`）互补：那个是"同月份不同机组"，本张是"同机组不同月份"。便于查看单台机组的 P90 TI-风速分布随月份的变化。

  图分上下两部分（3:1 高度比，共享 X 轴）：
  - **上方**：P90 TI-风速 bin 曲线，每个月份一条线，颜色按月份区分。
  - **下方**：**有效窗口数分组柱状图**，每个 bin 内每个月份一根柱，**颜色与上方曲线一一对应**（蓝线→蓝柱、橙线→橙柱）。柱子高度反映该 bin × 月份的有效数据量，便于判断上方 P90 值的样本可靠度——低柱对应 bin 的 P90 参考价值低。

**Excel**：`风场统计数据.xlsx`，包含以下 sheet：

- `日均{风速|湍流度|密度}` / `周均...` / `月均...`（按勾选项）
- `P90TI-全期` / `P90TI-月度_YYYY-MM` / `P90TI-周度_YYYY-MM-DD`

### 5.2 月度湍流表模式输出

**曲线图**（PNG）：月度湍流表模式输出两种互补视图：

- **bin 视图**（每风速 bin 一张）：`月度{bin}ms湍流度曲线.png`（如 `月度10ms湍流度曲线.png`）

  横轴为月份，纵轴为该 bin 的湍流度，每机组一条线。便于对比同一风速段下不同机组的差异。

- **机组视图**（每机组一张）：`机组{t}号月度湍流度曲线.png`（如 `机组22号月度湍流度曲线.png`）

  横轴为月份，纵轴为该机组的湍流度，每条线代表一个风速 bin。便于查看单台机组在不同风速段下的湍流度随月变化。

**Excel**：`风场统计数据.xlsx`，包含：

- `月度bin湍流度` sheet：列出全部机组/月份/bin/湍流度/样本数。

## 6. 处理逻辑说明

### 6.1 风速清洗（秒级原始数据模式，4 步）

1. 缺测值（≤ -999）置 NaN
2. 前后秒差 > 10 m/s 视为尖刺，置 NaN
3. 负值置 NaN
4. 超量程（> 50 m/s）置 NaN

### 6.2 TI 计算（秒级原始数据模式）

- 按设定的窗口/步长在清洗后的风速序列上滑动
- 窗口有效率达标 → 计算该窗口的 TI = std / mean（要求平均风速 > 0.5 m/s）
- 日均 TI = 当天所有有效窗口 TI 的均值

### 6.3 周/月聚合阈值

- 某机组某周数据天数 < 5 → 不出该周曲线
- 某机组某月数据天数 < 5 → 不出该月曲线

## 7. 常见问题

| 现象 | 可能原因与处理 |
|---|---|
| 启动后机组列表为空 | 默认未自动扫描，请先点「扫描」或切换数据类型 |
| 提示"未找到符合命名规则的数据文件" | 检查文件名是否严格匹配正则 `^(.+?)_(\d+)_(.+?)_(\d{4}-\d{2}-\d{2})_(\d{4}-\d{2}-\d{2})\.(csv\|xlsx)$` |
| 提示"未找到月度湍流文件" | 文件名需含 `月度湍流` 且为 `.xlsx` 后缀 |
| 某些文件被跳过 | 日志会标注原因：编码无法识别 / 无时间列 / 无风速列等 |
| 月/周曲线缺失某些机组 | 该机组该月/周有效天数 < 5 天，自动剔除 |
| 切换数据类型后日期范围不更新 | 数据类型切换会自动触发重扫；若仍未更新，手动点「扫描」 |
| 月度湍流模式勾选了风速/密度却没出图 | 此模式只输出月度 TI 时间序列，指标勾选在此模式下被忽略 |
| 缓存导致结果异常 | 删除 `输出\wind_farm\.cache.pkl` 后重新运行 |
| **风场对比** 扫描显示「缺 Excel: 风场X」 | 该子目录缺 `风场统计数据.xlsx`；从「风场数据统计」模块的输出目录拷过来 |
| **风场对比** 日志提示「缺 月均风速 sheet」 | 该 xlsx 是在「风场数据统计」未勾选「月」粒度时生成的，需重跑并勾选「月」 |
| **风场对比** 图里某风场只有部分月份 | 该风场数据本身覆盖范围短（如只有 5-12 月），符合预期，不补 0 |

## 9. 风场对比模块（wind_farm_compare）

把多个风场的月均风速 / 密度拉到一张图上横向对比，看哪个风场风况更好、密度差异等。

### 9.1 输入约定

输入目录（默认 `输入数据\wind_farm_compare\`）下**每个子目录 = 一个风场**：

```
输入数据\wind_farm_compare\
├── 风场A\
│   └── 风场统计数据.xlsx      ← 从「风场数据统计」模块的输出目录拷过来
├── 风场B\
│   └── 风场统计数据.xlsx
└── 风场C\
    └── 风场统计数据.xlsx
```

- **子目录名 = 风场名**：作为图例和 Excel 列名直接显示
- **必须包含 `风场统计数据.xlsx`**：由「风场数据统计」模块生成
- **Sheet 名要求**：`月均风速` / `月均密度`（也兼容 `月平均风速` / `月平均密度`）
- 隐藏目录（`.` 开头）会被忽略

> ⚠️ **重要**：`风场统计数据.xlsx` 必须在「风场数据统计」模块勾选**「月」粒度**的情况下生成，否则会缺月均 sheet。

### 9.2 聚合方式

**风场整体对比**（不是按机组）：

1. 取该风场 `月均风速` sheet 中所有机组的月均
2. 按月份 `groupby` 后简单平均，得到「该风场该月的全风场均值」
3. 每个风场一条线

### 9.3 面板操作

- **数据源**：路径 + 扫描 + 浏览（同风场数据统计）
- **参数**：
  - **对比指标**：风速 / 密度（多选，至少 1 个）
  - **日期范围**：全部 / 起止月份
- **扫描结果**：显示「找到 N 个风场（M 个就绪）」+ 风场名列表（缺 Excel 的单独标注）
- **日志反馈**：扫描/运行时缺数据会明确告诉用户，例如：
  ```
  [警告] 风场 风场C 缺 风场统计数据.xlsx，跳过
  [警告] 风场 风场B 月均 sheet 缺失或为空，跳过
           缺：月均风速, 月均密度
           请去「风场数据统计」模块勾选「月」粒度重跑该风场
  ```

### 9.4 输出结果

输出目录（默认 `输出\wind_farm_compare\`），运行时自动创建。

| 类型 | 文件名 | 说明 |
|---|---|---|
| PNG | `风场月均风速对比.png` | X=月份，每风场一条线，颜色自动分配 |
| PNG | `风场月均密度对比.png` | 同上 |
| Excel | `风场对比汇总.xlsx` | 每个 metric 一个 sheet：行=月份，列=风场名，值=月均 |

> 风场间月份范围不一致（如风场 B 只有 5-12 月数据）时，缺失月份留空，不补 0。

### 9.5 帮助文档

详细说明见 `src\help\wind_farm_compare.md`，可通过菜单「帮助 → ⚖ 风场对比」打开。

## 10. 叶片形状输出模块（shape_output）

三阶段风机叶片几何输出流水线。数据来源是叶片建模软件导出的 AppData 描述（`.geo` / `.prof` / 配置），最终输出 Focus.mac / STP 点云 / 多 sheet Excel。

### 10.1 输入约定

```
输入数据\shape_output\
└── AppData\
    ├── Aerofoil_coordinate\<族名>\*.prof    # 标准翼型族
    ├── Initialization\
```

**GEO 必须严格 7 列**：`展长(m)  弦长(m)  扭角(°)  相对厚度(%)  变桨轴(%)  预弯(m)  后掠(m)`。

### 10.2 三阶段流水线

| Tab | 阶段 | 主要动作 | 输出位置 |
|---|---|---|---|
| Stage1 | 基准翼型 | AppData + GEO → 计算外形 → 导出 AeroGEO 表 → 准备修正输入 | `stage1_base\AeroGEO\` |
| TE 修正 | 余弦过渡 | 改写 `GEO_for_correction.xlsx` 的 `TEth` 列（原地） | 同上 |
| Stage2 | 最终几何 | XFOIL TGAP/TSET 修正 → spapi 样条重建 → 导出最终文件 | `stage2_corrected\` |

**XFOIL 位置**：`src\_bin\xfoil.exe`（固定，不走设置对话框）。

### 10.3 TE 修正参数

| 参数 | 含义 |
|---|---|
| p1 = 修正区起始 | span/R ≥ p1 到叶尖前最后截面 = p2（mm） |
| p2 = 修正区厚度 | 目标尾缘总厚度 |
| p3 = 叶尖厚度 | 最后截面目标尾缘厚度 |
| p4 = 光顺过渡起始 | p4 → p1 之间用 `0.5*(1-cos(πt))` 平滑过渡 |

约束：**p4 < p1**。

### 10.4 Stage2 输出选项

| 选项 | 文件 |
|---|---|
| 标准翼型点云 | `standard_airfoil_points.xlsx` |
| 3D 叶片点云 | `blade_3d_points.xlsx` |
| 几何参数表 | `blade_aero_geometry.xlsx` |
| 修正后尾缘厚度 | `trailing_edge_thickness.xlsx`（含 Focus sheet） |
| Focus.mac | `Focus.mac` |
| STP 点云 | `3D_points.stp` |

## 11. 叶片结构套件模块（blade_converter）

FOCUS6 叶片结构分析的**一站式前后处理**：把原本散在独立工具里的 4 个子流程（blade_db 转换 / PRJ 更新 / Excel ↔ txt / 求解器调用）合并到一个模块。UI 采用 `QTabWidget` 四页（页签隐藏），由 banner 下方 **流水线 stepper** 切换阶段。

### 11.1 输入约定

```
输入数据\blade_converter\
├── blade_db.xlsx                 # TAB-1 截面属性数据库（17 列英文表头）
├── 变桨中心.xlsx（可选）          # TAB-1 变桨中心数据；留空则从 .mac 提取
├── blade_geometry.mac            # TAB-3 / TAB-4 FOCUS6 几何文件
├── <project>.prj                 # TAB-2 待更新的 FOCUS6 项目文件
├── focus2blade.xlsx              # TAB-2 输入（TAB-1 生成的，或自备同结构）
├── 载荷文件（可选）               # TAB-4 应变/挠度/载荷转化需要；7 列 x/fx/fy/fz/mx/my/mz
└── zspan 文件（可选）             # TAB-4 展向位置；留空时从 mac 的 PLACE SHAPE 自动提取
```

> `LOAD.LD1` 是 frbex 应变换算的**输出**而非输入；若你需要单独产生它，使用 TAB-4 单求解器中的「载荷转化」功能。

**blade_db.xlsx 关键列**（英文表头，区分大小写）：`Distance along blade` / `Chord` / `Twist angle` / `Mass per unit length` / `Flapwise stiffness` / `Edgewise stiffness` / `Neutral axis (x/y)` / `Center of elasticity (x/y)` / `Mass center (x/y)` / `Torsional stiffness` / `Extensional stiffness`。

### 11.2 求解器路径配置

**首次使用必须**先在「⚙ 设置」对话框底部「叶片结构套件 — 求解器路径」配置 **1 个**目录：FOCUS6 Modules 目录。

| 配置项 | 典型路径 |
|---|---|
| FOCUS6 Modules 目录 | `C:\Program Files (x86)\ECN_WMC\FOCUS6.3\Modules` |

该目录内含 `farob/` / `frbex/` / `utils/`（UserLoadcaseConverter）等子目录；TAB-4 按所选「求解器类型」自动选用对应子目录。不再需要分别配 3 个 `.exe`。

TAB-4 顶部会实时显示当前 Modules 目录（未配置时显示「(未配置)」并拒绝运行）。

### 11.3 四个 Tab 操作

| Tab | 功能 | 主要输入 | 主要输出 |
|---|---|---|---|
| TAB-1 | blade_db → focus2blade | `blade_db.xlsx`（+ 可选 `.mac`） | `focus2blade_<solver>.xlsx` |
| TAB-2 | PRJ 更新 | `<原>.prj` + `focus2blade.xlsx` | `<原>_updated.prj`（可选 `.prj.backup`） |
| TAB-3 | Excel ↔ txt 互转 | `blade_geometry.mac` 或 `blade_data.xlsx` | 对向格式 |
| TAB-4 | FOCUS6 求解器 | `.mac` + SUM 目录 + （按功能）载荷文件 + ZSPAN | SUM 目录下 FOCUS6 原生输出 |

**TAB-1 求解器类型**（`farob` / `frbex`）：影响输出文件名后缀（`_farob` / `_frbex`）。

**TAB-2 PRJ 更新**：默认输出到 `<原名>_updated.prj`，**不覆盖原文件**。勾选「运行前备份」额外生成 `<原名>.prj.backup`。17 个字段按 `PRJ_FIELD_MAPPING` 映射覆盖（`REF_X/REF_Y` / `CE_X/CE_Y` / `MASS` / `EIFLAP` / `GJ` / `EA` / `CS_X/CS_Y` 等）。

**TAB-3 双向**：
- 正向（mac → Excel）：解析 mac 中的 `DEF PARA / DEF SHAPE / POINTS / PLACE SHAPE / DEF MATERIAL / DEF S-N LINE / DEF LINE / DEF SECTION`，输出 7 个 sheet
- 反向（Excel → mac）：要求 xlsx 由正向生成（`PlaceShapes` 必须含 `CenterX / CenterY`）

**TAB-4 运行模式**：
- **单求解器**：只跑下方选定的单一功能（读取 mac / 解析 mac / 重量 / 频率 / 载荷转化 / 应变 / 叶尖挠度）
- **一键运行**：并行执行 [读取 mac + 载荷转化] → [解析 mac] → [重量 / 频率 / 应变 / 叶尖挠度]，任一步失败则后续不执行

**TAB-4 载荷文件 / ZSPAN 显隐规则**：
- 载荷文件（7 列 `x/fx/fy/fz/mx/my/mz`）：仅在功能 = 应变计算 / 叶尖挠度 / 载荷转化 时显示并校验
- ZSPAN 文件：仅在功能 = 应变计算 时显示；留空时从 mac 的 `PLACE SHAPE R_x.x <span_mm>` 行自动提取，生成 `zspan_auto.txt` 传入求解器
