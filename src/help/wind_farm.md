# 风场数据统计 — 帮助

> 模块 ID：`wind_farm`　　默认输入：`输入数据\wind_farm\`　　默认输出：`输出\wind_farm\`

本模块对风场秒级时间序列 / 月度湍流表进行批量统计与可视化，输出曲线图（PNG）+ 多 sheet Excel。

---

## 1. 支持的数据类型

两种类型独立处理，互不干扰——同一文件夹里同时存在两类文件时，按用户选中的数据类型只处理对应那一类。

| 类型 | 适用场景 | 来源 |
|---|---|---|
| 秒级原始数据 | 完整计算 TI / 风速 / 密度并按日/周/月聚合 | 智慧风场导出时间、风速、密度秒级数据 |
| 月度湍流表 | 直接出每风速 bin 的月度时间序列 | 智慧风场导出月度湍流度 |

---

## 2. 秒级原始数据

### 2.1 文件名规则（**必须严格匹配**）

正则：`^(.+?)_(\d+)_(.+?)_(\d{4}-\d{2}-\d{2})_(\d{4}-\d{2}-\d{2})\.(csv|xlsx)$`

模板：`风场名_机组号_机组号#_起始日期_结束日期.csv`（或 `.xlsx`）

| 段 | 含义 | 要求 |
|---|---|---|
| 风场名 | 任意文字 | 不含下划线 `_` |
| 机组号 | 整数 | 如 `010`、`21`（前导零可写可不写） |
| 机组号# | 机组号带 `#` 后缀 | 仅用于展示 |
| 起始日期 | 文件代表的"当天" | `YYYY-MM-DD`，**参与统计的日期以此为准** |
| 结束日期 | 次日（通常=起始+1 天） | 仅命名规范用 |

**示例**：

- `GDDLYQ_010_010#_2026-04-01_2026-04-02.csv` → GDDLYQ 风场、10 号机组、当天 2026-04-01
- `JNCYHQ_021_021#_2026-03-01_2026-03-02.xlsx` → JNCYHQ 风场、21 号机组、当天 2026-03-01

> 不匹配正则的文件会被跳过，日志提示 `[警告] 跳过无法解析的文件名: xxx`。

### 2.2 文件内容

**每个文件 = 一天的秒级时间序列**，约 86400 行（允许缺漏，按窗口有效率过滤）。至少包含时间列；风速列、密度列可选。

**列识别规则**：列名包含关键字即匹配（大小写不敏感），取第一个匹配列。

| 列类型 | 必需 | 识别关键字 | 数据要求 |
|---|---|---|---|
| 时间 | 必需 | `时间` / `time` / `日期` / `date` | `pd.to_datetime` 可解析 |
| 风速 | 选填 | `风速` / `wind` | 数值 m/s；缺测≤-999、尖刺、负值、>50 会被清洗 |
| 密度 | 选填 | `密度` / `density` | 数值 kg/m³ |

### 2.3 CSV 编码

按以下顺序尝试解码，首个成功的生效：

```
gbk → utf-8 → utf-8-sig → gb18030
```

全部失败则跳过该文件。XLSX 无编码问题。

### 2.4 风速清洗（4 步）

1. 缺测值（≤ -999）置 NaN
2. 前后秒差 > 10 m/s 视为尖刺，置 NaN
3. 负值置 NaN
4. 超量程（> 50 m/s）置 NaN

### 2.5 TI 计算

- 按设定的**窗口/步长**在清洗后的风速序列上滑动
- 窗口**有效率**达标 → 计算该窗口 TI = std / mean（要求平均风速 > 0.5 m/s）
- 日均 TI = 当天所有有效窗口 TI 的均值

### 2.6 周/月聚合阈值

- 某机组某周有效天数 < 5 → 不出该周曲线
- 某机组某月有效天数 < 5 → 不出该月曲线

## 3. 月度湍流表

### 3.1 文件名规则

`.xlsx` 文件，文件名包含关键字 `月度湍流`（大小写不敏感）。

示例：`江华金壁_月度湍流_1781775025621.xlsx`

### 3.2 文件内容

**每行 = 一个「机组 × 月份 × 风速 bin」的湍流度记录**。

| 列类型 | 必需 | 识别关键字 | 数据要求 |
|---|---|---|---|
| 机组号 | 必需 | `机位` / `机组` / `涡轮` | 整数 |
| 月份 | 必需 | `月` | 能解析为月份，取所属自然月 |
| 风速 bin 中心 | 必需 | `风速` / `bin` / `标签` | 整数 m/s（如 `10` = 9.5~10.5 段） |
| 湍流度数值 | 必需 | `数值` / `湍流` | 浮点（TI = std/mean） |
| 样本数 | 选填 | `样本` / `数据量` / `数量` / `count` | 整数；缺失按 1 计 |

> 多余列（风场名、拼音码等）自动忽略。前四项必需列不全则跳过该文件。

## 4. 面板操作

### 4.1 数据源

- **数据类型**：下拉切换 `秒级原始数据` / `月度湍流表`，自动重新扫描
- **路径**：当前输入目录（来自全局配置中心，初始为 `输入数据\wind_farm\`）
- **扫描**：手动扫描，**启动时不自动扫描**，必须先点此按钮或切换数据类型
- **浏览...**：选其他文件夹。**仅本次会话生效，重启后恢复**；永久修改请用「⚙ 设置」

### 4.2 统计内容（仅秒级原始数据模式有效）

- **指标**：风速 / 湍流度 / 密度（多选，至少 1 个）
- **粒度**：日 / 周 / 月（多选，至少 1 个）

### 4.3 机组配置

四列：**机组号 / 包含 / 高亮 / 叶型**

- **包含**：默认全选；可手动取消某些机组跳过统计。至少保留 1 个
- **高亮**：默认全不勾；勾选后该机组在曲线图中线宽加粗、不透明度提高
- **叶型**：手动输入文本，可留空。用于按叶型在图中区分机组线型
- 机组号列与各复选框不可编辑；仅叶型列可双击或按 F2 编辑

### 4.4 叶型-线型映射

机组表下方的表（标题旁带「刷新」按钮），为每种叶型指定图中曲线的线型 / 线宽 / 透明度。

- **触发刷新**：点 **刷新** 按钮或 **开始统计** 时自动刷新
- **表行来源**：自动收集叶型列去重；存在未填叶型的机组则多出一行 `(未指定)`
- **线型选项**：6 种（实线 / 密虚线 / 稀虚线 / 点划线 / 双点划线 / 点线）
- **线宽选项**：1.0 / 1.5 / 2.0 / 2.5 / 3.0 / 4.0
- **透明度选项**：0.3 / 0.4 / 0.5 / 0.6 / 0.7 / 0.8 / 0.9 / 1.0（默认 1.0）
- 默认按行序自动分配；用户选择后跨扫描保留
- **高亮模式下**：勾选高亮的机组用叶型设置的透明度；未勾选机组的透明度自动再 ×0.3 淡化，便于突出对比
- 已填叶型的机组：曲线使用该叶型对应线型；图例为 `#N号机组 (叶型名)`
- 未填叶型的机组：使用 `(未指定)` 行的线型；图例为 `#N号机组`（无后缀）

### 4.5 计算参数（仅秒级原始数据模式有效）

| 参数 | 含义 | 默认 | 范围 |
|---|---|---|---|
| 滑动窗口（分钟） | TI 计算窗口长度 | 10 | 1 ~ 60 |
| 滑动步长（分钟） | 窗口移动步长 | 10 | 1 ~ 30 |
| 有效率（%） | 窗口内有效数据占比阈值 | 90 | 0 ~ 100 |

### 4.6 日期范围

- 勾选 **全部日期**：使用扫描到的全部日期（默认）
- 取消勾选：手动选择起止日期（受扫描结果约束）

### 4.7 开始统计

点 **开始统计** 启动后台线程。按钮在处理期间变灰并显示「运行中...」，下方进度条与日志区实时刷新。完成后日志区显示生成图数与 Excel 路径。

## 5. 输出结果

输出落在该模块的输出目录（默认 `输出\wind_farm\`，可改），运行时自动创建。
同目录下 `.cache.pkl` 是读取缓存（按文件 mtime/size + TI 参数 + 缓存版本作键），可安全删除。

### 5.1 秒级原始数据模式

**曲线图**（PNG，按勾选的 指标 × 粒度 生成，最多 9 张）：

| 指标 | 日 | 周 | 月 |
|---|---|---|---|
| 风速 | 日平均风速曲线.png | 周平均风速曲线.png | 月平均风速曲线.png |
| 湍流度 | 日平均湍流度曲线.png | 周平均湍流度曲线.png | 月平均湍流度曲线.png |
| 密度 | 日平均空气密度曲线.png | 周平均空气密度曲线.png | 月平均空气密度曲线.png |

**P90 分布曲线**（仅勾选湍流度时生成；上下双轴，上为 P90 TI-风速曲线，下为有效窗口数分组柱状图）：

- `湍流度-风速分布P90曲线.png`（全期）
- `湍流度-风速分布P90曲线_YYYY-MM.png`（按月，需勾月粒度）
- `湍流度-风速分布P90曲线_周YYYY-MM-DD.png`（按周，需勾周粒度）

bin 划分：1 m/s 宽度、中心整数（9.5~10.5 → bin 10）；每 bin ≥ 10 个有效窗口才计算 P90。

**每机组按月对比图**（勾选「湍流度」+「月」粒度时生成，每机组一张）：

- `机组{t}号湍流度-风速分布P90曲线.png`
- 上方曲线每个月一条，下方柱状图颜色与曲线一一对应，便于判断各 bin 样本可靠度

**Excel**：`风场统计数据.xlsx`

- `日均{风速|湍流度|密度}` / `周均...` / `月均...`（按勾选项）
- `P90TI-全期` / `P90TI-月度_YYYY-MM` / `P90TI-周度_YYYY-MM-DD`

> 💡 此 xlsx 可被 **⚖ 风场对比** 模块复用：拷贝到 `输入数据\wind_farm_compare\{风场名}\` 下即可参与跨风场月均对比。要求运行时勾选「月」粒度，否则会缺 `月均风速` / `月均密度` sheet。

### 5.2 月度湍流表模式

**曲线图**（PNG）：两种互补视图。

- **bin 视图**（每风速 bin 一张）：`月度{bin}ms湍流度曲线.png`
  - 横轴=月份，纵轴=该 bin 的 TI，每机组一条线
- **机组视图**（每机组一张）：`机组{t}号月度湍流度曲线.png`
  - 横轴=月份，纵轴=该机组 TI，每条线代表一个风速 bin

**Excel**：`风场统计数据.xlsx`

- `月度bin湍流度` sheet：全部机组/月份/bin/湍流度/样本数

## 6. 常见问题

| 现象 | 处理 |
|---|---|
| 启动后机组列表为空 | 默认未自动扫描，请先点「扫描」或切换数据类型 |
| 提示"未找到符合命名规则的数据文件" | 检查文件名是否严格匹配正则 |
| 提示"未找到月度湍流文件" | 文件名需含 `月度湍流` 且为 `.xlsx` 后缀 |
| 某些文件被跳过 | 日志标注原因：编码无法识别 / 无时间列 / 无风速列等 |
| 月/周曲线缺失某些机组 | 该机组该月/周有效天数 < 5，自动剔除 |
| 月度模式勾选了风速/密度却没出图 | 此模式只输出月度 TI 时间序列，指标勾选被忽略 |
| 缓存导致结果异常 | 删除 `输出\wind_farm\.cache.pkl` 后重新运行 |
| 修改了路径但重启后丢失 | 面板内「浏览」仅会话内生效，永久修改请用「⚙ 设置」 |
